¿Qué es Python?Definición y filosofía

Python es un lenguaje de programación de alto nivel, interpretado, de propósito general y con tipado dinámico, diseñado con énfasis en la legibilidad del código y la productividad del programador.

Fue creado por Guido van Rossum y su primera versión fue publicada en 1991. El nombre no proviene de la serpiente, sino del grupo de comedia británico Monty Python's Flying Circus, que Van Rossum seguía durante el desarrollo.

La filosofía central de Python está resumida en el documento PEP 20, conocido como "El Zen de Python", que establece 19 principios de diseño.

>>> import this
The Zen of Python, by Tim Peters
Beautiful is better than ugly.
Explicit is better than implicit.
Simple is better than complex.
Readability counts.
>>>
Alto nivel

Se abstrae de los detalles del hardware. El programador se enfoca en la lógica, no en la memoria.

Interpretado

El código se ejecuta línea a línea por el intérprete. No requiere compilación previa.

Multiparadigma

Soporta programación imperativa, funcional y orientada a objetos de forma simultánea.

Open Source

Licencia libre. Comunidad global de millones de desarrolladores y paquetes.

"Python es un experimento en cómo construir un lenguaje que sea lo suficientemente potente para resolver problemas reales, pero lo suficientemente simple para aprenderse en una tarde." — Guido van Rossum, creador de Python

Historia y versionesEvolución del lenguaje

Python tiene más de tres décadas de historia continua, con una evolución constante que lo ha llevado de un proyecto personal a ser el lenguaje más popular del mundo.

1989

Inicio del proyecto

Guido van Rossum comienza a escribir Python en diciembre de 1989 en el Centrum Wiskunde & Informatica (CWI), Países Bajos, como sucesor del lenguaje ABC.

1991

Python 0.9.0

Primera versión pública. Ya incluía clases con herencia, manejo de excepciones, funciones y módulos.

2000

Python 2.0

Introducción de list comprehensions, recolector de basura y soporte Unicode. Usada durante 20 años.

2008

Python 3.0

Rediseño que rompió compatibilidad con Python 2. Corrigió inconsistencias fundamentales del lenguaje.

2020

Fin de Python 2

Fin oficial del soporte a Python 2.7. Toda la industria migró definitivamente a Python 3.

2025

Python 3.13+

Versión actual con mejoras en rendimiento, JIT experimental y el primer GIL-free build para múltiples hilos.

Características principalesPor qué Python es especial

Python combina simplicidad sintáctica con potencia real. Sus características no son accidentales, sino producto de decisiones de diseño deliberadas y coherentes.

CaracterísticaDescripciónBeneficio práctico
Sintaxis limpia Usa indentación como estructura en lugar de llaves {}. Código legible por humanos. Menos errores de formato.
Tipado dinámico No se declara el tipo de una variable. Se infiere en ejecución. Desarrollo más rápido. Código más corto.
Tipado fuerte No mezcla tipos implícitamente. "3" + 3 genera error. Menos bugs silenciosos. Comportamiento predecible.
Garbage collection Gestión automática de memoria mediante conteo de referencias. El programador no administra memoria manualmente.
Baterías incluidas Biblioteca estándar amplia que cubre casi cualquier tarea. Menos dependencias externas para proyectos comunes.
Multiplataforma El mismo código corre en Windows, macOS y Linux. Portabilidad total del código.
nota

La indentación obligatoria en Python no es solo estética: es parte de la gramática del lenguaje. Un bloque de código mal indentado es un error de sintaxis, lo que fuerza a todos los programadores a escribir código visualmente consistente.

Control de flujoCondicionales y ciclos

El control de flujo determina qué código se ejecuta y cuántas veces. Python usa la indentación para delimitar bloques, eliminando la necesidad de llaves o palabras clave de cierre.

Condicionales: if / elif / else

Python condicionales 
def clasificar_nota(nota):
    if nota >= 90:
        return "Excelente"
    elif nota >= 70:
        return "Aprobado"
    elif nota >= 60:
        return "Suficiente"
    else:
        return "Reprobado"

# Condicional en una línea (ternario)
estado = "mayor" if edad >= 18 else "menor"

# match-case: switch moderno (Python 3.10+)
match comando:
    case "salir":
        salir_programa()
    case "ayuda":
        mostrar_ayuda()
    case _:
        print("Comando desconocido")

Ciclos: for y while

Python bucles 
# for: itera sobre cualquier iterable
for i in range(5):           # 0, 1, 2, 3, 4
    print(i)

for letra in "Python":      # itera caracteres
    print(letra)

for idx, valor in enumerate(["a", "b", "c"]):
    print(f"{idx}: {valor}")

# while: itera mientras se cumple condición
contador = 0
while contador < 5:
    print(contador)
    contador += 1

# break, continue, else en bucles
for n in range(10):
    if n == 5:
        break             # sale del bucle
    if n % 2 == 0:
        continue          # salta a la siguiente iteración
    print(n)
else:                     # se ejecuta si el for terminó sin break
    print("Completado")

FuncionesDefinición, argumentos y lambdas

Las funciones son bloques de código reutilizables que reciben entradas, realizan operaciones y devuelven resultados. En Python son ciudadanos de primera clase: se pueden asignar a variables, pasar como argumentos y devolver desde otras funciones.

Python definición de funciones 
# Función básica con docstring
def saludar(nombre: str) -> str:
    """Devuelve un saludo personalizado."""
    return f"Hola, {nombre}!"

# Parámetros con valor por defecto
def potencia(base, exponente=2):
    return base ** exponente

print(potencia(3))       # 9  (usa exponente=2)
print(potencia(2, 8))    # 256

# *args: número variable de argumentos posicionales
def sumar(*numeros):
    return sum(numeros)

# **kwargs: argumentos con nombre
def perfil(**datos):
    for clave, valor in datos.items():
        print(f"{clave}: {valor}")

perfil(nombre="Ana", edad=25, ciudad="CDMX")

# Lambda: función anónima de una sola expresión
cuadrado = lambda x: x ** 2
doble    = lambda x: x * 2

numeros = [1, 2, 3, 4, 5]
pares   = list(filter(lambda x: x % 2 == 0, numeros))  # [2, 4]
tip

Las type hints (nombre: str, -> str) son opcionales pero muy recomendadas en Python 3. No afectan la ejecución, pero mejoran la documentación y permiten que herramientas como mypy detecten errores de tipos.

ColeccionesListas, tuplas, sets y diccionarios

Las colecciones son estructuras de datos que agrupan múltiples valores. Python tiene cuatro colecciones nativas con características bien diferenciadas.

Python list — mutable, ordenada, permite duplicados 
# Lista: la colección más usada
frutas = ["manzana", "banana", "mango"]

frutas.append("uva")      # agrega al final
frutas.insert(0, "kiwi")  # agrega en posición
frutas.remove("banana")  # elimina por valor
frutas.sort()            # ordena en lugar

# List comprehension: forma pythónica de crear listas
cuadrados = [x**2 for x in range(10)]
pares     = [x for x in range(20) if x % 2 == 0]

# Slicing: sub-listas
print(frutas[1:3])    # elementos 1 y 2
print(frutas[::-1])   # lista al revés
Python dict, tuple y set 
# Diccionario: clave-valor, ordenado (Python 3.7+)
persona = {
    "nombre": "Ana",
    "edad":   25,
    "ciudad": "CDMX"
}
persona["email"] = "ana@ejemplo.com"
edad = persona.get("edad", 0)   # valor por defecto si no existe

# Dict comprehension
cuadrados_d = {x: x**2 for x in range(5)}

# Tupla: inmutable, más rápida que lista
coordenadas = (19.43, -99.13)   # latitud, longitud CDMX
lat, lon    = coordenadas          # desempaquetado

# Set: colección sin duplicados, sin orden
colores  = {"rojo", "verde", "azul", "rojo"}  # rojo aparece una vez
colores2 = {"verde", "amarillo"}
comunes  = colores & colores2    # intersección
todos    = colores | colores2    # unión

Programación Orientada a ObjetosClases, herencia y métodos

Python soporta POO de forma completa. Una clase es una plantilla para crear objetos que encapsulan datos (atributos) y comportamiento (métodos).

Python clase, herencia y métodos especiales 
class Animal:
    """Clase base que representa un animal."""

    especie_dominante = "Homo sapiens"  # atributo de clase

    def __init__(self, nombre: str, sonido: str):
        self.nombre = nombre     # atributo de instancia
        self.sonido = sonido

    def hablar(self) -> str:
        return f"{self.nombre} dice: {self.sonido}!"

    def __str__(self) -> str:   # para print(objeto)
        return f"Animal({self.nombre})"

    def __repr__(self) -> str:  # representación técnica
        return f"Animal(nombre={self.nombre!r})"


# Herencia: Perro extiende Animal
class Perro(Animal):

    def __init__(self, nombre: str, raza: str):
        super().__init__(nombre, "Guau")  # llama al padre
        self.raza = raza

    def traer_pelota(self) -> str:
        return f"{self.nombre} trae la pelota."


# Uso
mi_perro = Perro("Rex", "Labrador")
print(mi_perro.hablar())          # Rex dice: Guau!
print(mi_perro.traer_pelota())   # Rex trae la pelota.
print(isinstance(mi_perro, Animal)) # True
clave

El primer parámetro de cualquier método de instancia es self, que hace referencia al objeto actual. No es una palabra reservada, es una convención universal: siempre se usa self.

Manejo de erroresExcepciones y depuración

En Python, los errores en tiempo de ejecución se llaman excepciones. El manejo de excepciones permite que el programa responda a situaciones inesperadas de forma controlada, en lugar de terminar abruptamente.

Python try / except / finally 
# Estructura completa de manejo de excepciones
try:
    numero   = int(input("Ingresa un número: "))
    resultado = 100 / numero

except ValueError:
    print("Error: Debes ingresar un número entero.")

except ZeroDivisionError:
    print("Error: No se puede dividir entre cero.")

except Exception as e:
    print(f"Error inesperado: {e}")

else:                          # se ejecuta si NO hubo excepción
    print(f"Resultado: {resultado}")

finally:                       # se ejecuta SIEMPRE
    print("Fin del programa.")


# Lanzar excepciones propias
class EdadInvalidaError(ValueError):
    pass

def validar_edad(edad: int):
    if edad < 0 or edad > 150:
        raise EdadInvalidaError(f"Edad {edad} fuera de rango.")
    return True

Aplicaciones reales¿Dónde se usa Python?

Python no es solo un lenguaje de aprendizaje: impulsa algunos de los sistemas más complejos del mundo. Su versatilidad lo hace dominante en sectores muy distintos.

Inteligencia Artificial

TensorFlow, PyTorch y Scikit-learn son los frameworks líderes de IA/ML, todos en Python. GPT, Gemini y otros LLMs se entrenan con código Python.

Ciencia de Datos

Pandas + NumPy + Matplotlib son el stack estándar de análisis de datos en empresas y academia. Jupyter Notebooks como entorno interactivo.

Desarrollo Web

Instagram, Pinterest y YouTube usan Django. Dropbox, Netflix y Uber usan Python en sus backends y microservicios.

Automatización

Scripts para tareas repetitivas, web scraping con Selenium/BeautifulSoup, bots, generación de reportes automáticos.

DevOps & Cloud

AWS, Google Cloud y Azure tienen SDKs en Python. Ansible, Salt y otras herramientas de infraestructura están escritas en Python.

Ciencia & Academia

La NASA, el CERN y el MIT usan Python para análisis de datos científicos, simulaciones y modelos computacionales.
