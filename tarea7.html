La Placa Arduino UNO

Conoce cada parte que conforma el microcontrolador más popular del mundo maker.

🔲

Microcontrolador ATmega328P

Es el cerebro de la placa. Un chip de 8 bits con 32KB de memoria Flash (donde vive tu programa), 2KB de SRAM (memoria de trabajo) y 1KB de EEPROM (datos permanentes). Opera a 16 MHz y gestiona todas las operaciones de entrada y salida.

// Núcleo principal
⚡

Pines Digitales (0–13)

14 pines que funcionan como entrada o salida digital. Los marcados con ~ (3, 5, 6, 9, 10, 11) soportan PWM para simular señales analógicas (control de brillo, velocidad de motores).

// INPUT / OUTPUT
📊

Pines Analógicos (A0–A5)

6 entradas analógicas con resolución de 10 bits (0–1023). Leen voltajes entre 0 y 5V provenientes de sensores como potenciómetros, fotorresistencias o sensores de temperatura.

// ADC 10-bit
🔌

Alimentación

Acepta 7–12V por el conector de barril, 5V directo por USB o por el pin VIN. Incorpora reguladores que proveen 5V y 3.3V estables para alimentar sensores y módulos externos con hasta 200mA.

// 5V / 3.3V / GND
🔁

Puerto USB (UART)

Permite cargar sketches desde la computadora y comunicarse en serie mediante el Monitor Serial del IDE de Arduino. Usa el chip CH340 o ATmega16U2 como puente USB-Serial.

// 9600 – 115200 baud
💡

LED Integrado (Pin 13)

LED soldado en placa conectado al pin digital 13. Ideal para pruebas rápidas de código sin necesitar hardware externo. Es el "Hola Mundo" del hardware: el clásico Blink.

// LED_BUILTIN
🔗

Pines I²C y SPI

I²C usa los pines A4 (SDA) y A5 (SCL) para conectar pantallas OLED, sensores ambientales y expansores de I/O con solo 2 cables. SPI usa los pines 10–13 para comunicación de alta velocidad con tarjetas SD, displays TFT y más.

// BUS DE DATOS

Componentes que se Conectan

Módulos y sensores que amplían las capacidades del Arduino.

💡

LED

Diodo emisor de luz. Se conecta a un pin digital a través de una resistencia de 220–470Ω para limitar la corriente. Fundamental para indicadores visuales de estado.

// pin digital + GND
🔘

Pulsador / Botón

Interruptor momentáneo. Se conecta entre un pin digital y GND, activando la resistencia interna pull-up con INPUT_PULLUP para evitar estados flotantes.

// INPUT_PULLUP
🌡️

Sensor de Temperatura (DHT11/22)

Mide temperatura y humedad relativa. Se conecta a un pin digital y requiere la librería DHT.h. El DHT22 ofrece mayor precisión y rango.

// 1-Wire protocol
📡

Sensor Ultrasónico (HC-SR04)

Mide distancias de 2 a 400 cm usando ultrasonido. Necesita 2 pines: TRIG (dispara pulso) y ECHO (recibe el rebote). Ideal para robots y detección de obstáculos.

// TRIG + ECHO
🖥️

Pantalla LCD 16×2

Display de 16 caracteres por 2 líneas. Se puede conectar con 6 pines en modo paralelo o con solo 2 pines usando el módulo adaptador I²C (dirección 0x27 o 0x3F).

// I²C / paralelo
📶

Módulo WiFi (ESP8266)

Permite conectar el Arduino a una red WiFi para IoT. Se comunica por serial (TX/RX) con comandos AT o se programa directamente con el IDE. Requiere alimentación de 3.3V estable.

// UART / AT commands
🔊

Buzzer Piezoeléctrico

Genera sonidos y melodías con la función tone(). El buzzer activo suena con solo aplicar voltaje; el pasivo necesita una señal PWM para generar distintas frecuencias.

// tone() / noTone()
⚙️

Servomotor SG90

Motor de posición precisa (0°–180°). Se controla con señal PWM usando la librería Servo.h. Alimentar desde fuente externa cuando se usan varios servos para no sobrecargar el Arduino.

// Servo.h / PWM
🔆

Fotorresistencia (LDR)

Sensor que varía su resistencia según la intensidad de luz. Se lee con un pin analógico en configuración divisor de voltaje con una resistencia de 10kΩ. Útil en sistemas de iluminación automática.

// analogRead()
Código de Ejemplo

Un sketch que lee temperatura y enciende un LED si supera el umbral.

// Lectura de sensor DHT11 con alerta LED
#include <DHT.h>

#define DHTPIN   2      // Sensor conectado al pin 2
#define DHTTYPE  DHT11
#define LED_ROJO 13     // LED de alerta en pin 13
#define UMBRAL   30.0   // Temperatura límite en °C

DHT dht(DHTPIN, DHTTYPE);

void setup() {
  Serial.begin(9600);
  pinMode(LED_ROJO, OUTPUT);
  dht.begin();
}

void loop() {
  float temp = dht.readTemperature();
  float hum  = dht.readHumidity();

  if (!isnan(temp)) {
    Serial.print("Temp: "); Serial.print(temp); Serial.println("°C");
    Serial.print("Hum:  "); Serial.print(hum);  Serial.println("%");

    digitalWrite(LED_ROJO, temp > UMBRAL ? HIGH : LOW);
  }

  delay(2000);  // Espera 2 segundos entre lecturas
}
¿Cómo se Utiliza Arduino?

Flujo de trabajo para pasar de una idea a un proyecto funcionando.

01

Instalar el IDE de Arduino

Descarga el Arduino IDE desde arduino.cc. Es gratuito y compatible con Windows, macOS y Linux. Incluye el editor de código, compilador y herramientas para subir programas a la placa por USB.

02

Conectar la placa por USB

Usa un cable USB tipo A–B (el mismo de las impresoras antiguas para UNO). El IDE detectará automáticamente el puerto COM/serial asignado. En Linux puede ser necesario añadir el usuario al grupo dialout.

03

Escribir el Sketch

Todo programa Arduino tiene dos funciones obligatorias: setup(), que se ejecuta una vez al inicio, y loop(), que se repite indefinidamente. En ellas controlas pines, lees sensores y defines la lógica de tu proyecto.

04

Compilar y Cargar

Presiona el botón ▶ (Subir) en el IDE. El código se compilará a lenguaje máquina para el ATmega328P y se transferirá a la memoria Flash de la placa. El LED TX/RX parpadeará durante la transferencia.

05

Monitorear y Depurar

Abre el Monitor Serial (Ctrl+Shift+M) para ver mensajes en tiempo real enviados por Serial.print(). Es la herramienta principal de depuración: muestra lecturas de sensores, variables y el estado del programa.

06

Escalar el Proyecto

Una vez que el código funciona, diseña un circuito permanente en una protoboard o PCB. Agrega librerías para módulos complejos, explora otras placas (Mega, Nano, MKR) y conecta módulos IoT para proyectos en red.
