// Fundamentos

¿Qué es HTML?

HTML (HyperText Markup Language) es el lenguaje estándar para crear páginas web. No es un lenguaje de programación — es un lenguaje de marcado, lo que significa que usa etiquetas para definir la estructura y el contenido de un documento.

Cada vez que abres un sitio web, tu navegador (Chrome, Firefox, Safari) lee el código HTML y lo convierte en la página visual que ves. HTML es la base sobre la que se construye todo lo demás: CSS da estilos y JavaScript agrega interactividad.

📄

Lenguaje de Marcado

Usa etiquetas para describir el significado y estructura del contenido, no la lógica.

🌐

Estándar W3C

Mantenido por el World Wide Web Consortium. HTML5 es la versión actual desde 2014.

🔗

HyperText

Los hipervínculos conectan documentos entre sí, creando la "tela" de la web.

🧱

Base del Trío Web

HTML = estructura, CSS = diseño, JavaScript = comportamiento. Sin HTML no hay web.

HTML no es programar: es como construir los cimientos y las paredes de una casa antes de pintarla o poner la electricidad.

// Estructura

Anatomía de un documento

Todo documento HTML sigue la misma estructura base. Conocerla es el primer paso para crear cualquier página web.

estructura-basica.html
<!-- 1. Declaración del tipo de documento -->
<!DOCTYPE html>

<!-- 2. Elemento raíz -->
<html lang="es">

  <!-- 3. Cabecera (invisible en el navegador) -->
  <head>
    <meta charset="UTF-8"/>
    <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
    <title>Título de mi página</title>
    <link rel="stylesheet" href="styles.css"/>
  </head>

  <!-- 4. Cuerpo (todo lo visible) -->
  <body>
    <h1>Mi primera página web</h1>
    <p>Este es un párrafo de texto.</p>
  </body>

</html>
01

<!DOCTYPE html>

Le dice al navegador que el documento usa HTML5. Siempre va en la primera línea. No es una etiqueta HTML, es una declaración.

02

<html lang="es">

Elemento raíz que envuelve todo el documento. El atributo lang indica el idioma — importante para accesibilidad y SEO.

03

<head>

Contiene metainformación: charset, viewport, título, enlaces a CSS y fuentes. Nada aquí se muestra visualmente en la página.

04

<body>

Todo el contenido visible va aquí: textos, imágenes, botones, videos, formularios. Es el "cuerpo" de la página.

// Etiquetas de texto

Títulos, párrafos & texto

Las etiquetas de texto son las más usadas. Definen jerarquía de contenido, párrafos, énfasis, citas y formatos especiales.

<h1> — <h6> Encabezados

6 niveles de encabezado. <h1> es el más importante (solo uno por página para SEO), <h6> el menor.

<h1>Título principal</h1>
<h2>Subtítulo</h2>
<p> Párrafo

Define un párrafo de texto. El navegador agrega espacio automático arriba y abajo de cada párrafo.

<p>Texto del párrafo.</p>
<strong> Énfasis fuerte

Texto en negrita con peso semántico. Indica importancia. Diferente a <b> que solo es visual.

<strong>Importante</strong>
<em> Énfasis

Texto en cursiva con énfasis semántico. Los lectores de pantalla cambian el tono de voz aquí.

<em>énfasis</em>
<br/> Salto de línea

Inserta un salto de línea sin crear un nuevo párrafo. Es una etiqueta vacía (no tiene cierre).

Línea 1<br/>
Línea 2
<blockquote> Cita en bloque

Para citas largas de otras fuentes. El atributo cite indica la URL de la fuente original.

<blockquote cite="url">
  Cita aquí
</blockquote>
<span> Contenedor inline

Contenedor genérico en línea para aplicar estilos CSS a porciones de texto sin romper el flujo.

<span class="rojo">texto</span>
<code> Código inline

Para mostrar fragmentos de código dentro de un párrafo. Se muestra en fuente monoespaciada.

<code>console.log()</code>

// Etiquetas semánticas

HTML5 Semántico

HTML5 introdujo etiquetas semánticas: etiquetas que describen el significado del contenido, no solo su apariencia. Esto mejora el SEO, la accesibilidad y la legibilidad del código.

Antes de HTML5 todo se hacía con <div> sin significado. Ahora cada sección tiene su propia etiqueta con semántica clara.

layout-semantico.html
<body>

  <!-- Cabecera del sitio -->
  <header>
    <nav>
      <a href="/">Inicio</a>
      <a href="/about">Acerca de</a>
    </nav>
  </header>

  <!-- Contenido principal -->
  <main>
    <article>
      <h1>Título del artículo</h1>
      <section>
        <p>Sección del artículo...</p>
      </section>
    </article>

    <aside>
      <p>Contenido relacionado</p>
    </aside>
  </main>

  <!-- Pie de página -->
  <footer>
    <p>&copy; 2024 Mi Sitio</p>
  </footer>

</body>
<header>Cabecera

Encabezado de la página o de una sección. Suele contener el logo, título del sitio y navegación.

<header>Logo + Nav</header>
<nav>Navegación

Bloque de navegación principal. Los lectores de pantalla lo identifican como menú de navegación.

<nav><a>Enlace</a></nav>
<main>Principal

Contenido principal de la página. Solo debe haber uno por documento. No incluye nav, header ni footer.

<main>Contenido</main>
<section>Sección

Agrupa contenido temáticamente relacionado dentro de la página. Debe tener su propio encabezado.

<section><h2>Tema</h2></section>
<article>Artículo

Contenido independiente que podría existir por sí solo: un blog post, noticia o comentario.

<article>Post completo</article>
<aside>Lateral

Contenido relacionado pero secundario: barras laterales, publicidad, datos curiosos, información extra.

<aside>Info extra</aside>
<footer>Pie de página

Pie de la página o de una sección. Contiene copyright, enlaces legales, contacto, créditos.

<footer>© 2024</footer>
<div>Contenedor genérico

Contenedor genérico de bloque sin semántica. Úsalo solo cuando ninguna etiqueta semántica aplique.

<div class="container"></div>

// Listas y enlaces

Listas, Links e Imágenes

Los enlaces, listas e imágenes son los elementos más comunes en cualquier página web. Dominar estos tres grupos te permite construir el 80% de cualquier sitio.

listas-enlaces.html
<!-- Listas no ordenadas (viñetas) -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Listas ordenadas (numeradas) -->
<ol>
  <li>Aprender HTML</li>
  <li>Aprender CSS</li>
</ol>

<!-- Enlace (hipervínculo) -->
<a href="https://google.com"
   target="_blank"
   rel="noopener">
  Visitar Google
</a>

<!-- Imagen -->
<img
  src="foto.jpg"
  alt="Descripción de la imagen"
  width="400"
  height="300"
/>
EtiquetaAtributo claveDescripción
<a>href="url"URL a la que enlaza. Puede ser relativa o absoluta.
<a>target="_blank"Abre el enlace en una nueva pestaña del navegador.
<a>rel="noopener"Seguridad: impide que la nueva página acceda a la original.
<img>src="ruta"Ruta de la imagen (local o URL). Obligatorio.
<img>alt="texto"Texto alternativo para lectores de pantalla y si la imagen falla. Obligatorio.
<ul>type="disc|circle|square"Tipo de viñeta (se recomienda usar CSS en su lugar).
<ol>start="n"Número inicial de la lista ordenada.
<li>value="n"Valor específico del ítem en listas ordenadas.

// Formularios

Formularios e Inputs

Los formularios permiten al usuario enviar información. Son la base del registro, login, búsqueda y contacto en cualquier sitio. HTML5 amplió los tipos de <input> con validación nativa del navegador.

formulario.html
<form action="/enviar" method="POST">

  <!-- Campo de texto -->
  <label for="nombre">Nombre:</label>
  <input type="text" id="nombre"
         name="nombre" required
         placeholder="Tu nombre"/>

  <!-- Correo electrónico (validación automática) -->
  <input type="email" name="email"
         placeholder="correo@ejemplo.com"/>

  <!-- Número con rango -->
  <input type="number" min="1" max="100"/>

  <!-- Lista desplegable -->
  <select name="grado">
    <option value="1">1° Semestre</option>
    <option value="2">2° Semestre</option>
  </select>

  <!-- Área de texto -->
  <textarea rows="4" cols="40"
             placeholder="Mensaje..."></textarea>

  <!-- Botón enviar -->
  <button type="submit">Enviar</button>

</form>

// Tablas y multimedia

Tablas, Video & Audio

HTML5 integró soporte nativo para audio y video, eliminando la necesidad de Flash. Las tablas sirven para datos tabulares — nunca para diseño de layout.

tabla-multimedia.html
<!-- TABLA con encabezados -->
<table>
  <thead>
    <tr>
      <th>Lenguaje</th>
      <th>Año</th>
      <th>Uso</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>HTML</td>
      <td>1993</td>
      <td>Estructura</td>
    </tr>
  </tbody>
</table>

<!-- VIDEO nativo HTML5 -->
<video
  src="video.mp4"
  width="640"
  controls
  autoplay
  muted
  loop
  poster="miniatura.jpg">
  Tu navegador no soporta video.
</video>

<!-- AUDIO nativo HTML5 -->
<audio src="musica.mp3" controls>
</audio>

<!-- IFRAME para incrustar contenido -->
<iframe
  src="https://www.youtube.com/embed/ID"

  allowfullscreen>
</iframe>

// Atributos globales

Atributos Esenciales

Los atributos globales pueden usarse en cualquier etiqueta HTML. Son fundamentales para conectar HTML con CSS y JavaScript.

classclass="btn rojo"Nombre(s) de clase para aplicar estilos CSS. Puede repetirse en varios elementos.

// Buenas prácticas

Código Correcto

Escribir HTML correcto no solo hace que el sitio funcione — también lo hace accesible, rápido en cargarse y bien posicionado en Google.

  • Siempre cierra tus etiquetas. Las etiquetas de bloque como <div>, <p>, <h1> siempre necesitan su cierre </tag>. Las vacías como <img> y <br> se cierran con />.
  • Usa minúsculas. HTML no distingue mayúsculas, pero <div> es la convención estándar, no <DIV>.
  • Indenta el código. Usa 2 o 4 espacios para cada nivel de anidación. Hace el código legible.
  • El atributo alt en imágenes es obligatorio. Sin él, los usuarios con discapacidad visual no saben qué muestra la imagen.
  • Solo un <h1> por página. Los motores de búsqueda usan el H1 para entender el tema principal. Usa <h2>-<h6> para el resto.
  • Usa etiquetas semánticas. Prefiere <header>, <main>, <footer> sobre <div> genérico cuando el contenido lo justifica.
  • Separa HTML, CSS y JS. No pongas estilos con style="" en línea ni JavaScript con onclick="". Usa archivos separados.
  • Valida tu código. Usa el validador oficial de la W3C en validator.w3.org para verificar que no tienes errores.
  • Agrega charset UTF-8. Evita que los acentos y la ñ se vean como símbolos extraños en todos los navegadores.
  • Optimiza tus imágenes. Usa formatos modernos como WebP y agrega atributos width y height para evitar saltos de diseño al cargar.

El buen HTML no se nota: cuando está bien hecho, la página simplemente funciona para todos, en todos los dispositivos, en todos los navegadores.
